# Example Queries for Ethos

## Project Management

- What did we decide about the API design?
- When is the deadline for Phase 2?
- Who's responsible for the frontend redesign?
- What are the main blockers mentioned this week?
- What's the status of the authentication feature?

## Technical Discussions

- Why did we choose PostgreSQL over MongoDB?
- How do we handle authentication in the API?
- What's our current deployment strategy?
- What libraries did we decide to use for the frontend?
- How are we managing state in the application?

## Team Coordination

- Who's working on the frontend?
- What did Sarah say about the design mockups?
- When did John mention the code review?
- Who volunteered to fix the bug in production?
- What did the team decide in yesterday's standup?

## Business Decisions

- What was the outcome of the pricing discussion?
- Why did we pivot to the new feature?
- What feedback did we get from the client?
- What are the Q4 priorities?
- When do we plan to launch the beta?

## Troubleshooting

- How did we fix the performance issue last time?
- What was the solution to the CORS error?
- How do we handle rate limiting?
- What's the workaround for the third-party API issue?
- How did we resolve the deployment failure?

## Process Questions

- What's our code review process?
- How do we handle hotfixes?
- What's the release schedule?
- How do we prioritize features?
- What's our testing strategy?

## Tips for Best Results

1. **Be specific**: "What did Sarah decide about the user dashboard?" is better than "What did Sarah say?"

2. **Include context**: "Why did we choose React over Vue for the frontend?" works better than "Why React?"

3. **Use names**: "What did John suggest for the API?" helps narrow down results

4. **Include timeframes**: "What did we decide in last week's meeting?" is more specific

5. **Ask about decisions**: Questions about decisions and conclusions work best

6. **Avoid real-time info**: Ethos searches history, not live data or future plans

## When Ethos Says "I Don't Know"

This means:
- The information wasn't discussed in indexed channels
- The keywords don't match any conversations
- The discussion happened before messages were indexed
- Try rephrasing with different terms
